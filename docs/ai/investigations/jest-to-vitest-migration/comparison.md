# Jest vs Vitest 比較

## 実行時間の比較

| 統計 | Jest | Vitest | 差分 |
|---|---|---|---|
| 最小値 | 7695ms | 6346ms | **-1349ms (-17.5%)** |
| 最大値 | 21836ms | 13740ms | **-8096ms (-37.1%)** |
| 中央値 | 9066.5ms | 7641.5ms | **-1425ms (-15.7%)** |
| 平均値 | 10445.5ms | 8216.4ms | **-2229ms (-21.3%)** |
| 標準偏差 | 4189.4ms | 2079.0ms | **-2110ms (-50.4%)** |

※ 両者ともに外れ値が1件存在（Jest Run5: 21836ms、Vitest Run6: 13740ms）

## 改善された項目

### 実行時間
- すべての統計値で Vitest が高速。
- 中央値で 1425ms（約15%）短縮。
- 外れ値の最大値も 21836ms → 13740ms と改善。
- **標準偏差が 50% 削減**（4189ms → 2079ms）: 実行時間のばらつきが大幅に安定。

### 設定の簡潔さ
- `transformIgnorePatterns` が不要に: unified, remark, hast 等の ESM only パッケージのために長い正規表現を維持する必要がなくなった。
- `next/jest.js` ラッパーが不要に: Vite が TypeScript・JSX・Next.js を直接処理。
- CSS モック（`styleMock.js` + `moduleNameMapper`）が `css: false` 一行に置き換え。

### 依存パッケージ
- `jest`, `jest-environment-jsdom`, `ts-jest`, `@types/jest` の4パッケージを削除。
- `vitest` はすでに Storybook の `@storybook/addon-vitest` 用にインストール済みだったため追加なし。

## 改善されなかった項目

### 実行時間（絶対値）
- Vitest も中央値 7641ms と、1秒未満には程遠い。起動コストが支配的。
- 外れ値は引き続き発生（13740ms）。

### テストコードの変更工数
- 31ファイルのテストファイルに手動変更が必要だった（`jest.*` → `vi.*` 等）。

## 結論

移行により実行時間・安定性の両面で改善。特に設定の簡潔さ（ESM 対応の複雑な設定が不要）が大きなメリット。
