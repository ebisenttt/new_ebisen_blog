# 依存パッケージアップデートガイド

## このドキュメントの目的
依存パッケージをアップデートする際の注意事項と手順をまとめる。

## セキュリティ前提条件
2025年後半からnpmサプライチェーン攻撃が流行している。作業開始前に
`pnpm-workspace.yaml` が存在し、以下の項目が設定されていることを確認すること。
設定が不足している場合はユーザーの承認を得てから追加すること。

- `onlyBuildDependencies`: postInstallの実行を許可するパッケージ一覧
- `strictDepBuilds: true` — 未承認スクリプトがある場合に `pnpm install` をエラーにする
- `minimumReleaseAge: 1440` — リリース後24時間未満のパッケージをブロックする
- `blockExoticSubdeps: true` — gitリポジトリやtarball URLからの推移的依存をブロックする
- `trustPolicy: no-downgrade` — 信頼レベルが低下したパッケージをブロックする

## 手順
1. `pnpm outdated` でアップデート候補を一覧確認する
2. `pnpm audit` でセキュリティ上の問題があるパッケージを確認する
3. アップデート対象と順序の計画を立てる（依存関係の競合に注意）
4. アップデートを実行する。競合で更新できないパッケージはユーザーに報告して判断を求める
5. `pnpm typecheck` で型エラーがないことを確認する
6. `pnpm test` でテストがパスすることを確認する
7. `pnpm build` でビルドが通ることを確認する
8. 完了をユーザーに報告する
