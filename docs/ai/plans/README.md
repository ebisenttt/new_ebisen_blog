# ai/plans/

AIエージェントへの作業依頼計画書を格納する。

## フロー

1. ユーザーが `tmp.md` として計画の草案を作成する
2. AIエージェントが `tmp.md` を読み込み、ユーザーと相談しながら計画を精査する
3. AIエージェントが `tmp.md` を適切なファイル名に変更する
4. 計画が確定したらAIエージェントが計画に沿って実装を進める

## ファイル命名規則

ブランチ命名規則のprefixに準拠し、ケバブケースで命名する。

```
{type}-{description}.md
```

| type | 用途 |
|---|---|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `chore` | 依存更新・設定変更など |
| `docs` | ドキュメント整備 |
| `ci` | CI/CD変更 |
| `refactor` | リファクタリング |

**例**: `feat-add-dark-mode.md`, `chore-upgrade-typescript.md`, `fix-broken-layout.md`

## 注意

- 草案段階のファイル名は常に `tmp.md`
- `tmp.md` が既に存在する場合は上書きしてよい（作業中の計画は1つを想定）
- 完了した計画書は履歴として残す（削除しない）
